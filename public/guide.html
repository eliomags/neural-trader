System Overview

🎯 What is Neural Trader?

Neural Trader is an advanced, AI-powered trading platform that combines machine learning, technical analysis, and automated execution to trade both cryptocurrencies and stocks. The system uses LSTM neural networks, ensemble learning, and sophisticated risk management to make intelligent trading decisions.

Key Components

🧠 Neural Network Engine

LSTM-based deep learning models that analyze market patterns and predict price movements with ensemble voting for improved accuracy.

📊 Technical Analysis

20+ technical indicators including RSI, MACD, Bollinger Bands, Moving Averages, and custom proprietary indicators.

💱 Multi-Exchange Support

Connects to multiple exchanges: Binance, Coinbase, Kraken for crypto; Alpaca, Interactive Brokers for stocks.

⚡ Real-Time Execution

WebSocket connections for live market data, instant order execution, and real-time portfolio updates.

🛡️ Risk Management

Kelly Criterion position sizing, correlation analysis, stop-loss/take-profit, and maximum drawdown protection.

📈 Performance Analytics

Sharpe ratio, win rate, profit factor, maximum drawdown, and comprehensive trade history tracking.

Quick Start Guide

✅ Default Mode: Paper Trading

The system starts in Paper Trading Mode by default. This means:

  • No real money is at risk
  • You can test all features safely
  • Perfect for learning and strategy testing
  • Uses simulated market data and executions

Getting Started in 5 Minutes

1 Start the Application

./start.sh

This will install dependencies, build the project, and start the server on port 3001.

2 Access the Dashboards

  • Crypto Trading: http://localhost:3001
  • Stock Trading: http://localhost:3001/stocks.html
  • This Guide: http://localhost:3001/guide.html

3 Explore Paper Trading

Click "Start Trading" to begin paper trading. The system will:

  • Generate simulated market data
  • Make trading decisions using AI models
  • Execute paper trades
  • Track performance metrics

4 Monitor Performance

Watch the dashboards update in real-time with:

  • Portfolio value and P&L
  • Open positions
  • Trading signals
  • Performance metrics

Live Trading Setup ADVANCED

⚠️ Important Warning

Live trading involves real money and financial risk. Only proceed if you:

  • Understand the risks involved in automated trading
  • Have tested thoroughly in paper trading mode
  • Can afford to lose the capital you're trading with
  • Have proper API keys from supported exchanges

Transitioning to Live Trading

1 Obtain API Keys

Get API keys from your chosen exchanges (see API Setup section below).

2 Configure Environment

Edit the .env file in the project root:

# Set to 'live' for real trading TRADING_MODE=live # Add your API keys (see API Setup section) BINANCE_API_KEY=your_key_here BINANCE_SECRET_KEY=your_secret_here # ... more keys

3 Set Risk Parameters

Configure risk management in config/trading.json:

{ "maxPositionSize": 0.1, // Max 10% per position "maxDrawdown": 0.2, // Stop at 20% drawdown "stopLossPercent": 0.05, // 5% stop loss "takeProfitPercent": 0.15 // 15% take profit }

4 Start with Small Capital

Begin with a small amount to verify everything works correctly.

5 Monitor Closely

Watch the system carefully during the first few days of live trading.

Troubleshooting

Common Issues

🔴 Port Already in Use

Error: "Error: listen EADDRINUSE: address already in use :::3001"

Solution:

# Find and kill the process using port 3001 lsof -i :3001 kill -9 [PID] # Or use a different port PORT=3002 npm start

🔴 WebSocket Connection Failed

Error: "WebSocket connection failed"

Solution:

  • Check if you're in paper trading mode (no real connections needed)
  • Verify API keys are correct
  • Check exchange API status
  • Ensure IP is whitelisted on exchange

🔴 Module Not Found

Error: "Cannot find module..."

Solution:

# Clear and reinstall dependencies rm -rf node_modules package-lock.json npm install npm run build

🔴 Database Errors

Error: "Database connection failed"

Solution:

# Reset database rm -rf data/*.db npm start # Will recreate database

Performance Issues

  • High CPU Usage: Normal during model training. Reduce concurrent pairs being traded.
  • Memory Leaks: Restart the application daily. Check for zombie WebSocket connections.
  • Slow Response: Reduce the number of indicators or trading pairs.

Frequently Asked Questions

Q: Is this safe to use with real money?

A: The system includes multiple safety features, but all trading carries risk. Start with paper trading, then small amounts if you choose to go live. Never trade money you can't afford to lose.

Q: How much can I expect to earn?

A: Returns vary greatly based on market conditions, settings, and luck. The system aims for consistent risk-adjusted returns, not maximum profits. Past performance doesn't guarantee future results.

Q: Can I modify the trading strategies?

A: Yes! The code is open and modifiable. Key strategy files are in src/strategies/. Be sure to test changes in paper trading first.

Q: What's the minimum capital needed?

A: For paper trading: $0. For live crypto: $100+ recommended. For live stocks: $1,000+ recommended (due to PDT rules if under $25,000).

Q: Does it work on weekends?

A: Crypto trading works 24/7 including weekends. Stock trading only works during market hours (Mon-Fri, 9:30 AM - 4:00 PM ET).

Q: How do I add more trading pairs?

A: Edit config/trading.json and add symbols to the tradingPairs array.

Q: Can I run this on a Raspberry Pi?

A: Yes, but performance may be limited. Recommend at least 4GB RAM and reducing the number of trading pairs.

Q: Is my data secure?

A: API keys are stored locally in your .env file. Never share this file or commit it to version control. The system doesn't send data to external servers.

Q: How do I update the system?

A: Pull the latest changes from the repository:

git pull origin main npm install npm run build ./start.sh

Q: Can I run multiple instances?

A: Yes, but use different ports and separate API keys to avoid conflicts.
